# Описание проекта

Проект использует паттерн MVC (Model-View-Controller) для обработки запросов и отображения контента.

## Program.cs

В файле Program.cs происходит настройка и запуск приложения. В частности там указаны настройки БД для SQLite и есть закомментированные настройки для PostrgeSQL.

Есть пример жизненных циклов в виде настройки 3-х севисов 
```csharp
builder.Services.AddTransient<TransientTime>();
builder.Services.AddScoped<ScopedTime>();
builder.Services.AddSingleton<SingletonTime>();
```

Посмотреть работу можно после запуска по адресу https://localhost:7126/services

Указан шаблон путей по умолчанию для простых контроллеров

```csharp
app.MapControllerRoute(
    name: "default",
    pattern: "/{controller=Home}/{action=Index}/{id?}");
```

Есть настройки для Swagger по пути https://localhost:7126/swagger
Там можно тестировать api

## appsettings.json

`appsettings.json` - файл для указания паременных конфигурации проекта.
В секции `ConnectionStrings` указаны строки для подключения к БД.
Секция `MySettings` отвечает за хранение конфигурационных данных и при старте приложения данные из нее попадают в класс `MySettings`. Этот класс лежит по пути `Data/MySettings.cs`.

Получать данные можно через DI `IOptions<MySettings> options`, лежит в `options.Value`.

## Папка Contollers

В данной папке лежат контроллеры без Area. Они обрабатывает запросы и готовят ответ

## Контекст базы данных Data/ApplicationDbContext.cs

В данном файле указаны списки сущностей бд, их связи друг с другом.

Указание сущности(таблицы) происходит через запись:
```csharp
public DbSet<SimpleEntity> SimpleEntities { get; set; }
```
Указывать дополнительные связи, ограничения и т.п. можно в методе `OnModelCreating`. В проекте он минимальный и больше выполняет информационную цель о своем существовании.

После изменения данного класса, нужно создать миграцию:
```bash
dotnet ef migrations add <Имя миграции>
```
После этой команды создадутся новые файлы в папке `Migrations`. Применять миграции для БД SQLite не нужно. И прменение уже включено при старте проекта.

Посмотреть пример работы с контекстом для получения данных можно в `Controllers/EntitiesController.cs` и `Areas/Controllers/EntitiesController.cs`

## Папка Models

В данной папке лежат модели как для базы данных, так и для вывода на страницу пользователя - ViewModels. Логики в моделях нет, только свойства, для передачи данных.

## Папка Services

В данной папке лежат сервисы, которые нужно переиспользовать в разных контроллерах. Обычно их помещают в DI и получают из него при выполнении запроса. Стоит быть аккуратным при использовании уровня зависимостей. По умолчанию можно ставить `Transient`.

Сейчас там леат только сервисы для примера. После начала работы, и можно будет удалить.

## Папка Views

В данной папке лежат частичные представления - cshtml файлы. Они отвечают за генерацию html с подстановкой данных внутрь.

`_ViewStart.cshtml` - отвечает за то, какой layout будет использоваться по умолчанию. Трогать не нужно.

`_ViewImports.cshtml` - отвечает за импорты namespace по умолчанию для всех вьюшек. Трогать не нужно.

`Shared/_Layout.cshtml` - layout по умолчанию. Трогать только при желании изменить внешний вид сайта по умолчанию.

Папка `Shared` - отвечает за хранение вьющек доступных для всех контроллеров.

Название прочих папок должно совпадать с именем контроллера, а название представления внутри с названием метода контроллера. Т.е. для метода `Index` в `HomeController.cs` вьюшка должна лежать в папке `Home` и называться `Index.cshtml`.

## Папка wwwroot

Папка для хранения статики. Все, что находится в данной папке, будет досутпно при запуске. Например `wwwroot/favicon.ico` будет доступен по адресу https://localhost:7126/favicon.ico

## Папка Areas

В данной папке обычно находятся отдельные области проекта, как админки, апи и т.п. Обычно внутри создают папку с названием области и в данной папке уже создают папки `Controllers` и `Views`. Хранение вьюшек в отдельной подпаке области в данном случае обязательное.
